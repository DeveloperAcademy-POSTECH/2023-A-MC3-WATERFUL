


################
# 제목(필수)
# <타입>: <제목>의 형식으로 커밋 메시지 작성
# 1. 제목은 변경사항이 무엇인지 명확히 작성
# 2. 끝에 마침표 미작성
# 3. '~했음' 대신 수행한 작업을 명시 (ex. '~작업', '~작성' 등)
# ex) feat: 로그인 기능 추가

################
# Body(선택)
# Body는 제목과 한 개의 공백을 두고 작성
# 작업한 내용을 구체적으로 설명하거나, 다중 작업인 경우 사용
# 여러 줄의 내용을 작성할 때에는 '-' 혹은 '1.' 등으로 줄 구분

################
# Footer(선택)
# 현재 커밋과 관련된 이슈 번호 등을 Footer에 작성

################
# 작업 종류
# feat : 새로운 기능 추가
# fix : 버그 수정
# doc : 문서 수정
# refactor : 코드 리팩토링
# style : 코드 의미에 영향을 주지 않는 변경사항
# chore : 빌드 부분 혹은 패키지 매니저 및 기타 수정사항
